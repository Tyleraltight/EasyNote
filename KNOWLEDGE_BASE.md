# 项目避坑指南 (Knowledge Base)

本文件用于记录项目开发过程中遇到的坑、失败经验及反思，以持续演进并避免重蹈覆辙。

## 📍 已知陷阱记录

### 1. 代理与网络环境问题带来的耗时 (2026-03-26)
- **触发情境**：在尝试寻找或拉取 GitHub 上的远程文档时，由于没有挂载代理工具，持续执行了多轮 `git clone` 与 `git pull`。
- **导致的后果**：终端反复抛出 SSL/TLS 连接失败报错（如 `OpenSSL SSL_connect: Connection was reset in connection to github.com:443`），白白浪费了大量时间进行无意义的重试与探测。
- **避坑原则 / 反思**：
  1. **不要盲目探测**：如需同步远程 GitHub 仓库状态，必须先确认当前是否具备相应的网络代理环境。如果网络受限，应果断放弃命令行拉取网络资源的动作。
  2. **聚焦本地资源**：优先解决工作区内已经拉取下来且明确要求修改的本地文件。
  3. **遇到疑点立即提问**：若对“两个文件”或“目标修改位置”存在认知模糊，不要擅自过度搜索和跨项目排查，严格执行“需求模糊时必须提问澄清”的工程纪律。
